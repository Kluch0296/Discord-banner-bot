# Инструкция по миграции на версию 2.0

## Что изменилось

Версия 2.0 переходит на мультисерверную модель с базой данных. Теперь:
- ✅ Каждый сервер имеет свои настройки
- ✅ Настройки хранятся в БД SQLite (`jail_bot.db`)
- ✅ `config.json` содержит только токен и префикс
- ✅ Настройка через удобную панель `/jail-config`

## Шаги миграции

### 1. Сохраните старый конфиг

Сделайте резервную копию вашего текущего `config.json`:
```bash
cp config.json config.json.backup
```

### 2. Обновите config.json

Замените содержимое `config.json` на:
```json
{
  "bot_token": "ВАШ_ТОКЕН_ЗДЕСЬ",
  "command_prefix": "!"
}
```

### 3. Запустите бота

```bash
python bot.py
```

При первом запуске автоматически создастся база данных `jail_bot.db`.

### 4. Настройте каждый сервер

На каждом сервере, где используется бот:

1. Используйте команду `/jail-config`
2. Настройте все разделы:
   - ⚙️ Основное (админские роли)
   - 📍 Каналы и роли
   - ⏱️ Сроки ареста
   - ⚖️ Апелляции
3. Нажмите **💾 Сохранить**

### 5. Перенос настроек из старого конфига

Используйте данные из `config.json.backup` для заполнения панели настроек:

**Из старого конфига:**
```json
{
  "jail_channel_id": 123456789,
  "jail_role_id": 987654321,
  "admin_role_ids": [111111111, 222222222],
  "arrest_notification_channel_id": 333333333,
  "appeal_voting_channel_id": 444444444,
  "arrest_durations": [...],
  "appeal_voting_durations": {...}
}
```

**В новую панель:**
1. Откройте `/jail-config`
2. В разделе "Каналы и роли" выберите соответствующие каналы и роли
3. В разделе "Основное" выберите админские роли
4. В разделе "Сроки ареста" добавьте пресеты из `arrest_durations`
5. В разделе "Апелляции" настройте время голосования из `appeal_voting_durations`
6. Сохраните настройки

## Важные замечания

### База данных

Файл `jail_bot.db` создается автоматически и содержит все настройки серверов. 

**Рекомендации:**
- Делайте резервные копии `jail_bot.db` регулярно
- Не удаляйте этот файл, иначе потеряете все настройки
- Добавьте `jail_bot.db` в `.gitignore` если используете git

### Конфиг

Новый `config.json` содержит только:
- `bot_token` - токен бота
- `command_prefix` - префикс для текстовых команд (по умолчанию `!`)

**Рекомендации:**
- Добавьте `config.json` в `.gitignore`
- Используйте `config.example.json` как шаблон для других разработчиков

### Префикс команд

Префикс теперь **НЕ настраивается** для каждого сервера отдельно - он глобальный для всех серверов. Если вам нужны разные префиксы на разных серверах, используйте только slash-команды.

## Проверка миграции

После настройки проверьте работу бота:

1. ✅ Команда `/jail-config` открывает панель
2. ✅ В панели отображаются все настроенные каналы и роли
3. ✅ Команда `!арест` работает в голосовом канале
4. ✅ Арест выполняется корректно
5. ✅ Апелляции работают (если настроены)

## Откат на старую версию

Если что-то пошло не так:

1. Остановите бота
2. Восстановите старый `config.json`:
   ```bash
   cp config.json.backup config.json
   ```
3. Замените файлы бота на старую версию
4. Запустите бота

## Поддержка

При возникновении проблем:
1. Проверьте логи бота в консоли
2. Убедитесь, что все настройки в `/jail-config` заполнены
3. Проверьте права бота на сервере
4. Убедитесь, что роль бота выше роли заключенного
