# Discord Бот для Ареста Участников

Discord бот на Python для временного ареста участников голосовых каналов с управлением ролями.

## Возможности

- 🚔 Арест участников голосового канала по команде администратора
- ⏱️ Выбор времени ареста (от 30 секунд до 1 часа)
- 🔒 Автоматическое управление ролями (снятие всех ролей и выдача роли заключенного)
- 📍 Перемещение в специальный канал "тюрьмы"
- ⏰ Автоматическое освобождение по истечении времени
- 🔓 Возможность досрочного освобождения администратором

## Требования

- Python 3.8 или выше
- discord.py 2.3.0 или выше

## Установка

1. Клонируйте или скачайте файлы проекта

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Настройте конфигурацию в файле `config.json`:
```json
{
  "bot_token": "YOUR_BOT_TOKEN_HERE",
  "jail_channel_id": 123456789012345678,
  "jail_role_id": 123456789012345678,
  "admin_role_ids": [123456789012345678, 987654321098765432],
  "command_prefix": "!"
}
```

### Получение необходимых ID

#### Bot Token (Токен бота):
1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Создайте новое приложение или выберите существующее
3. Перейдите в раздел "Bot"
4. Нажмите "Reset Token" и скопируйте токен
5. Вставьте токен в поле `bot_token` в `config.json`

#### Jail Channel ID (ID канала тюрьмы):
1. В Discord включите режим разработчика: Настройки → Расширенные → Режим разработчика
2. Создайте голосовой канал для "тюрьмы" (например, "🔒 Тюрьма")
3. Нажмите правой кнопкой на канал → Копировать ID
4. Вставьте ID в поле `jail_channel_id` в `config.json`

#### Jail Role ID (ID роли заключенного):
1. Создайте роль для заключенных (например, "Заключенный")
2. Настройте права роли (рекомендуется ограничить доступ к каналам)
3. Нажмите правой кнопкой на роль в настройках сервера → Копировать ID
4. Вставьте ID в поле `jail_role_id` в `config.json`

#### Admin Role IDs (ID админских ролей):
1. Определите, какие роли на сервере должны иметь доступ к командам бота
2. Для каждой роли: нажмите правой кнопкой на роль в настройках сервера → Копировать ID
3. Добавьте все ID в массив `admin_role_ids` в `config.json`
4. Пример: `"admin_role_ids": [123456789012345678, 987654321098765432]`
5. **Важно:** Если массив пустой `[]`, бот будет проверять стандартные права администратора Discord

### Настройка прав бота

При добавлении бота на сервер, убедитесь что у него есть следующие права:
- ✅ Управление ролями (Manage Roles)
- ✅ Перемещать участников (Move Members)
- ✅ Просмотр каналов (View Channels)
- ✅ Отправка сообщений (Send Messages)
- ✅ Чтение истории сообщений (Read Message History)

**ВАЖНО:** Роль бота должна быть выше роли заключенного и всех ролей, которые он будет снимать с участников!

## Использование

### Команда ареста

Пользователь с одной из админских ролей (или с правами администратора, если `admin_role_ids` пуст) должен находиться в голосовом канале и ввести команду:
```
!арест
```

Бот выведет сообщение "Кого арестовать?" с кнопками всех участников текущего голосового канала (кроме самого администратора и ботов).

После выбора участника появится сообщение "На какой срок?" с кнопками:
- 30 секунд
- 60 секунд
- 3 минуты
- 5 минут
- 15 минут
- 1 час

После выбора времени:
1. У участника снимаются все роли
2. Выдается роль заключенного
3. Участник перемещается в канал тюрьмы
4. Запускается таймер

По истечении времени:
1. Роль заключенного снимается
2. Возвращаются все прежние роли
3. Участник перемещается обратно в исходный голосовой канал

### Команда досрочного освобождения

Пользователь с админской ролью может досрочно освободить участника:
```
!освободить @участник
```

## Обработка ошибок

- Если пользователь не в голосовом канале: "❌ Вы не в голосовом чате!"
- Если в канале нет других участников: "❌ В голосовом канале нет других участников для ареста!"
- Если у пользователя нет необходимых прав: "❌ У вас нет прав для использования этой команды!"

## Технические детали

### Основные компоненты:

- **MemberSelectView**: UI компонент для выбора участника
- **TimeSelectView**: UI компонент для выбора времени ареста
- **arrest_member()**: Функция выполнения ареста
- **release_member_after_timeout()**: Асинхронная функция освобождения по таймеру

### Хранение данных:

Информация об арестованных участниках хранится в словаре `arrested_users`:
```python
{
    member_id: {
        'original_channel': VoiceChannel,
        'original_roles': List[Role],
        'jail_role': Role,
        'guild': Guild
    }
}
```

## Устранение неполадок

### Бот не отвечает на команды
- Проверьте, что токен бота правильный
- Убедитесь, что у бота есть права на чтение сообщений
- Проверьте префикс команды в `config.json`

### Бот не может переместить участника
- Убедитесь, что роль бота выше роли участника
- Проверьте права "Перемещать участников"

### Бот не может управлять ролями
- Убедитесь, что роль бота выше всех ролей, которыми он управляет
- Проверьте права "Управление ролями"

### Неправильные ID в конфиге
- Включите режим разработчика в Discord
- Скопируйте ID заново
- ID должны быть числами без кавычек
