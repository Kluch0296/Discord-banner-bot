# Discord Бот для Ареста Участников

Discord бот на Python для временного ареста участников голосовых каналов с управлением ролями и мультисерверной поддержкой.

## Возможности

- 🚔 Арест участников голосового канала по команде администратора
- ⏱️ Настраиваемые пресеты времени ареста
- 🔒 Автоматическое управление ролями (снятие всех ролей и выдача роли заключенного)
- 🔐 **Автоматическая настройка прав доступа** - роль заключённого блокируется от всех голосовых каналов кроме тюрьмы
- 📍 Перемещение в специальный канал "тюрьмы"
- ⏰ Автоматическое освобождение по истечении времени
- 🔓 Возможность досрочного освобождения администратором
- ⚖️ **Система апелляций с голосованием сообщества**
- 🗳️ Публичное голосование за/против освобождения заключенного
- ⏲️ Динамическое время голосования в зависимости от срока ареста
- 🌐 **Мультисерверная поддержка** - каждый сервер имеет свои настройки
- ⚙️ **Удобная панель настроек** через slash-команду `/jail-config`
- 💾 **База данных SQLite** для хранения настроек серверов

## Требования

- Python 3.8 или выше
- discord.py 2.3.0 или выше

## Установка

1. Клонируйте или скачайте файлы проекта

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `config.json` на основе `config.example.json`:
```json
{
  "bot_token": "YOUR_BOT_TOKEN_HERE",
  "command_prefix": "!"
}
```

4. Настройте бота через панель настроек (см. раздел "Настройка через панель")

### Создание и настройка бота в Discord Developer Portal

#### 1. Создание приложения

1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Нажмите **"New Application"**
3. Введите имя приложения и нажмите **"Create"**

#### 2. Настройка Bot

1. Перейдите в раздел **"Bot"** в левом меню
2. Нажмите **"Reset Token"** и скопируйте токен
3. Вставьте токен в поле `bot_token` в `config.json`

#### 3. Включение Privileged Gateway Intents

В разделе **"Bot"** прокрутите вниз до **"Privileged Gateway Intents"** и включите:

- ✅ **PRESENCE INTENT** - для отслеживания статусов пользователей (опционально)
- ✅ **SERVER MEMBERS INTENT** - **ОБЯЗАТЕЛЬНО** для работы с участниками сервера
- ✅ **MESSAGE CONTENT INTENT** - **ОБЯЗАТЕЛЬНО** для чтения содержимого сообщений (система апелляций)

**ВАЖНО:** Без `SERVER MEMBERS INTENT` и `MESSAGE CONTENT INTENT` бот не сможет работать!

#### 4. Настройка OAuth2 и прав бота

1. Перейдите в раздел **"OAuth2"** → **"URL Generator"**
2. В **"SCOPES"** выберите:
   - ✅ `bot` - базовые права бота
   - ✅ `applications.commands` - для slash-команд

3. В **"BOT PERMISSIONS"** выберите следующие права:

**Общие права:**
- ✅ **Manage Roles** (Управление ролями) - для снятия/выдачи ролей
- ✅ **Manage Channels** (Управление каналами) - для создания каналов и настройки прав доступа
- ✅ **View Channels** (Просмотр каналов) - для доступа к каналам

**Текстовые каналы:**
- ✅ **Send Messages** (Отправка сообщений) - для уведомлений
- ✅ **Manage Messages** (Управление сообщениями) - для удаления сообщений апелляций
- ✅ **Read Message History** (Чтение истории сообщений) - для работы с сообщениями

**Голосовые каналы:**
- ✅ **Move Members** (Перемещать участников) - для перемещения в тюрьму

4. Скопируйте сгенерированную ссылку и используйте её для добавления бота на сервер

#### 5. Добавление бота на сервер

1. Откройте сгенерированную ссылку в браузере
2. Выберите сервер из списка
3. Нажмите **"Authorize"** (Авторизовать)
4. Подтвердите права бота

#### 6. Настройка иерархии ролей на сервере

**КРИТИЧЕСКИ ВАЖНО:**

После добавления бота на сервер, в настройках сервера (**Server Settings** → **Roles**):

1. Роль бота должна быть **выше** роли заключенного
2. Роль бота должна быть **выше** всех ролей, которые он будет снимать с участников
3. Если роль бота ниже - он не сможет управлять этими ролями!
(или можно просто дать ему админские права на сервере)

Пример правильной иерархии:
```
👑 Владелец сервера
🤖 Роль бота          ← должна быть здесь
👮 Администратор
👤 Участник
🔒 Заключенный        ← ниже роли бота
@everyone
```

### Минимальные требуемые права

Для корректной работы бота необходимы следующие права:

**Обязательные Intents (в Developer Portal):**
- ✅ **SERVER MEMBERS INTENT** - работа с участниками
- ✅ **MESSAGE CONTENT INTENT** - чтение сообщений для апелляций

**Обязательные права бота (Bot Permissions):**
- ✅ **Manage Roles** - управление ролями участников
- ✅ **Manage Channels** - создание каналов, ролей и настройка прав доступа к голосовым каналам
- ✅ **Move Members** - перемещение в голосовых каналах
- ✅ **View Channels** - просмотр каналов
- ✅ **Send Messages** - отправка уведомлений
- ✅ **Manage Messages** - удаление сообщений
- ✅ **Read Message History** - чтение истории

**Опциональные Intents:**
- ⚪ **PRESENCE INTENT** - не требуется для работы бота

## Настройка через панель

### Открытие панели настроек

Администратор сервера должен использовать slash-команду:
```
/jail-config
```

Откроется интерактивная панель настроек с текущим состоянием конфигурации и кнопками разделов.

### Разделы настроек

#### ⚙️ Основное
- **Админские роли** - роли, которые могут использовать команды бота
  - Выбираются через Role Select (можно выбрать несколько ролей)
  - Если не настроены, проверяются стандартные права администратора Discord

#### 📍 Каналы и роли
- **Канал тюрьмы** 🔒 - голосовой канал, куда перемещаются арестованные
  - Можно выбрать существующий канал или создать новый через кнопку "➕ Создать канал тюрьмы"
- **Роль заключенного** 👤 - роль, которая выдается арестованным
  - Можно выбрать существующую роль или создать новую через кнопку "➕ Создать роль заключенного"
- **Канал уведомлений** 📢 - текстовый канал для уведомлений об арестах
  - Можно выбрать существующий канал или создать новый через кнопку "➕ Создать канал уведомлений"
- **Канал голосования** ⚖️ - текстовый канал для апелляций
  - Можно выбрать существующий канал или создать новый через кнопку "➕ Создать канал голосования"

Все каналы и роли выбираются через удобные селекты Discord или создаются автоматически с возможностью ввода названия.

**Создание каналов и ролей:**
- При нажатии на кнопку создания откроется модальное окно для ввода названия
- Каналы и роли создаются с предустановленными названиями по умолчанию, которые можно изменить
- После создания канал/роль автоматически выбирается в настройках
- Для создания требуется право "Manage Channels" у бота

#### ⏱️ Сроки ареста
Управление пресетами времени ареста:
- **➕ Добавить пресет** - создать новый вариант времени
- **✏️ Изменить пресет** - отредактировать существующий
- **🗑️ Удалить пресет** - удалить ненужный вариант

Каждый пресет содержит:
- **Название** (например, "5 минут")
- **Длительность в секундах** (например, 300)

#### ⚖️ Апелляции
Настройка времени голосования для каждого пресета ареста:
- **✏️ Изменить время голосования** - настроить для конкретного пресета
- **🔄 Установить значения по умолчанию** - автоматически рассчитать время

Время голосования:
- `0` секунд = апелляция недоступна
- `> 0` секунд = апелляция доступна с указанным временем голосования

### Сохранение настроек

После внесения изменений:
1. Нажмите **💾 Сохранить** для применения изменений
2. Или **🔄 Сбросить** для отмены несохраненных изменений
3. **❌ Закрыть** для выхода из панели (только если нет несохраненных изменений)

Панель показывает предупреждение, если есть несохраненные изменения.

**При сохранении настроек автоматически:**
- 🔐 Настраиваются права доступа роли заключённого ко всем голосовым каналам
- ✅ Роль заключённого получает доступ только к каналу тюрьмы
- ❌ Доступ ко всем остальным голосовым каналам блокируется
- 📝 Доступ к текстовым каналам остаётся без изменений

### Валидация настроек

Перед сохранением проверяется:
- ✅ Все обязательные каналы настроены
- ✅ Роль заключенного настроена
- ✅ Админские роли настроены
- ✅ Все каналы и роли существуют на сервере
- ✅ Пресеты времени корректны (нет дублей, положительные значения)
- ✅ Настройки апелляций соответствуют пресетам

## Использование

### Команда ареста

Пользователь с админской ролью должен находиться в голосовом канале и ввести команду:
```
!арест
```

Процесс:
1. Бот выводит кнопки со всеми участниками канала
2. Админ выбирает участника
3. Бот показывает кнопки с настроенными пресетами времени
4. Админ выбирает время ареста
5. Участник арестовывается

После ареста:
1. У участника снимаются все роли
2. Выдается роль заключенного
3. Участник перемещается в канал тюрьмы
4. Отправляется уведомление в канал уведомлений
5. Запускается таймер

По истечении времени:
1. Роль заключенного снимается
2. Возвращаются все прежние роли
3. Участник перемещается обратно в исходный голосовой канал

### Команда досрочного освобождения

Пользователь с админской ролью может досрочно освободить участника:
```
!освободить @участник
```

### Система апелляций

После ареста участник получает уведомление в канале уведомлений:

```
@участник, вас арестовали по решению @администратор.

Вы можете подать апелляцию. Время голосования: X секунд.
[Кнопка: Подать апелляцию]
```

#### Процесс подачи апелляции:

1. **Нажатие на кнопку "Подать апелляцию"**
   - Кнопка доступна только арестованному пользователю
   - Другие пользователи не могут нажать на кнопку

2. **Ввод текста апелляции**
   - После нажатия кнопки появляется сообщение: "Введите текст апелляции:"
   - Участник отправляет сообщение с причиной апелляции
   - Сообщение автоматически удаляется
   - Появляется статус: "Апелляция на рассмотрении..."

3. **Голосование**
   - Апелляция отправляется в канал голосования
   - Любой участник сервера может проголосовать
   - Можно переголосовать (последний голос учитывается)
   - Заключенные не могут голосовать

4. **Подсчет голосов**
   - По истечении времени голосования подсчитываются голоса
   - Если голосов "за" >= голосов "против" - заключенный освобождается
   - Если голосов "против" больше - заключенный остается под арестом
   - Результаты отображаются в сообщении с апелляцией

#### Ограничения:

- Одна апелляция на один арест
- Апелляция может быть недоступна для коротких сроков (настраивается)
- Только арестованный может подать апелляцию
- Голосовать могут все участники сервера, кроме заключенных

## Мультисерверная поддержка

Каждый сервер имеет свою независимую конфигурацию:
- ✅ Свои каналы и роли
- ✅ Свои пресеты времени ареста
- ✅ Свои настройки апелляций
- ✅ Свои админские роли

Настройки хранятся в базе данных SQLite (`jail_bot.db`) и автоматически создаются при первом использовании команды `/jail-config` на сервере.

## Устранение неполадок

### Бот не отвечает на команды
- Проверьте, что токен бота правильный
- Убедитесь, что у бота есть права на чтение сообщений
- Проверьте префикс команды в `config.json`

### Slash-команда /jail-config не работает
- Убедитесь, что бот имеет право "Use Application Commands"
- Проверьте, что у вас есть права администратора на сервере
- Перезапустите бота для синхронизации команд

### Бот не может переместить участника
- Убедитесь, что роль бота выше роли участника
- Проверьте права "Перемещать участников"
- Убедитесь, что канал тюрьмы настроен в `/jail-config`

### Бот не может управлять ролями
- Убедитесь, что роль бота выше всех ролей, которыми он управляет
- Проверьте права "Управление ролями"
- Убедитесь, что роль заключенного настроена в `/jail-config`

### Панель настроек не открывается
- Проверьте, что у вас есть права администратора
- Убедитесь, что бот запущен и онлайн
- Проверьте логи бота на наличие ошибок

### Ошибка "Настройки не найдены"
- Откройте `/jail-config` - настройки создадутся автоматически
- Проверьте, что файл `jail_bot.db` не поврежден

## Миграция со старой версии

Если вы использовали старую версию бота с `config.json`:

1. Сохраните старый `config.json` как резервную копию
2. Создайте новый `config.json` только с `bot_token` и `command_prefix`
3. Запустите бота
4. Используйте `/jail-config` для настройки каждого сервера
5. Перенесите настройки из старого конфига вручную через панель
