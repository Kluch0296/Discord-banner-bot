# Discord Бот для Ареста Участников

Discord бот на Python для временного ареста участников голосовых каналов с управлением ролями.

## Возможности

- 🚔 Арест участников голосового канала по команде администратора
- ⏱️ Выбор времени ареста (от 30 секунд до 1 часа)
- 🔒 Автоматическое управление ролями (снятие всех ролей и выдача роли заключенного)
- 📍 Перемещение в специальный канал "тюрьмы"
- ⏰ Автоматическое освобождение по истечении времени
- 🔓 Возможность досрочного освобождения администратором
- ⚖️ **Система апелляций с голосованием сообщества**
- 🗳️ Публичное голосование за/против освобождения заключенного
- ⏲️ Динамическое время голосования в зависимости от срока ареста

## Требования

- Python 3.8 или выше
- discord.py 2.3.0 или выше

### Получение необходимых ID

#### Bot Token (Токен бота):
1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Создайте новое приложение или выберите существующее
3. Перейдите в раздел "Bot"
4. Нажмите "Reset Token" и скопируйте токен
5. Вставьте токен в поле `bot_token` в `config.json`

#### Jail Channel ID (ID канала тюрьмы):
1. В Discord включите режим разработчика: Настройки → Расширенные → Режим разработчика
2. Создайте голосовой канал для "тюрьмы" (например, "🔒 Тюрьма")
3. Нажмите правой кнопкой на канал → Копировать ID
4. Вставьте ID в поле `jail_channel_id` в `config.json`

#### Jail Role ID (ID роли заключенного):
1. Создайте роль для заключенных (например, "Заключенный")
2. Настройте права роли (рекомендуется ограничить доступ к каналам)
3. Нажмите правой кнопкой на роль в настройках сервера → Копировать ID
4. Вставьте ID в поле `jail_role_id` в `config.json`

#### Admin Role IDs (ID админских ролей):
1. Определите, какие роли на сервере должны иметь доступ к командам бота
2. Для каждой роли: нажмите правой кнопкой на роль в настройках сервера → Копировать ID
3. Добавьте все ID в массив `admin_role_ids` в `config.json`
4. Пример: `"admin_role_ids": [123456789012345678, 987654321098765432]`
5. **Важно:** Если массив пустой `[]`, бот будет проверять стандартные права администратора Discord

#### Arrest Notification Channel ID (ID канала уведомлений об аресте):
1. Создайте текстовый канал для уведомлений об арестах (например, "🔔 Аресты")
2. Нажмите правой кнопкой на канал → Копировать ID
3. Вставьте ID в поле `arrest_notification_channel_id` в `config.json`
4. В этом канале арестованные будут получать уведомления и смогут подать апелляцию

#### Appeal Voting Channel ID (ID канала голосования по апелляциям):
1. Создайте текстовый канал для голосования (например, "⚖️ Апелляции")
2. Нажмите правой кнопкой на канал → Копировать ID
3. Вставьте ID в поле `appeal_voting_channel_id` в `config.json`
4. В этом канале будут появляться апелляции для публичного голосования

#### Arrest Durations (Варианты времени ареста):
Массив объектов, определяющих доступные варианты времени ареста:
```json
"arrest_durations": [
  {
    "label": "Текст на кнопке",
    "seconds": количество_секунд
  }
]
```
- `label` - текст, который будет отображаться на кнопке выбора времени
- `seconds` - длительность ареста в секундах

**Примеры:**
- `{"label": "30 секунд", "seconds": 30}` - арест на 30 секунд
- `{"label": "5 минут", "seconds": 300}` - арест на 5 минут (300 секунд)
- `{"label": "1 час", "seconds": 3600}` - арест на 1 час (3600 секунд)

Вы можете добавлять, удалять или изменять варианты по своему усмотрению.

#### Appeal Voting Durations (Время голосования по апелляциям):
Словарь, определяющий длительность голосования для каждого срока ареста:
```json
"appeal_voting_durations": {
  "секунды_ареста": секунды_голосования
}
```
- Ключ - длительность ареста в секундах (в виде строки)
- Значение - длительность голосования в секундах (число)
- Если значение `0` - апелляция недоступна для данного срока

**Примеры:**
- `"30": 0` - для ареста на 30 секунд апелляция недоступна
- `"60": 15` - для ареста на 60 секунд голосование длится 15 секунд
- `"3600": 120` - для ареста на 1 час голосование длится 2 минуты

**Важно:** Ключи в `appeal_voting_durations` должны соответствовать значениям `seconds` из `arrest_durations`.

### Настройка прав бота

При добавлении бота на сервер, убедитесь что у него есть следующие права:
- ✅ Управление ролями (Manage Roles)
- ✅ Перемещать участников (Move Members)
- ✅ Просмотр каналов (View Channels)
- ✅ Отправка сообщений (Send Messages)
- ✅ Чтение истории сообщений (Read Message History)

**ВАЖНО:** Роль бота должна быть выше роли заключенного и всех ролей, которые он будет снимать с участников!

## Использование

### Команда ареста

Пользователь с одной из админских ролей (или с правами администратора, если `admin_role_ids` пуст) должен находиться в голосовом канале и ввести команду:
```
!арест
```

Бот выведет сообщение "Кого арестовать?" с кнопками всех участников текущего голосового канала (кроме самого администратора и ботов).

После выбора участника появится сообщение "На какой срок?" с кнопками:
- 30 секунд
- 60 секунд
- 3 минуты
- 5 минут
- 15 минут
- 1 час

После выбора времени:
1. У участника снимаются все роли
2. Выдается роль заключенного
3. Участник перемещается в канал тюрьмы
4. Запускается таймер

По истечении времени:
1. Роль заключенного снимается
2. Возвращаются все прежние роли
3. Участник перемещается обратно в исходный голосовой канал

### Команда досрочного освобождения

Пользователь с админской ролью может досрочно освободить участника:
```
!освободить @участник
```

### Система апелляций

После ареста участник получает уведомление в канале уведомлений об аресте:

```
@участник, вас арестовали по решению @администратор.

Вы можете подать апелляцию. Время голосования: X секунд.
[Кнопка: Подать апелляцию]
```

#### Процесс подачи апелляции:

1. **Нажатие на кнопку "Подать апелляцию"**
   - Кнопка доступна только арестованному пользователю
   - Другие пользователи не могут нажать на кнопку

2. **Ввод текста апелляции**
   - После нажатия кнопки появляется сообщение: "Введите текст апелляции:"
   - Участник отправляет сообщение с причиной апелляции
   - Сообщение автоматически удаляется
   - Появляется статус: "Апелляция на рассмотрении..."

3. **Голосование**
   - Апелляция отправляется в канал голосования
   - Формат сообщения:
     ```
     Апелляция от @участник
     
     Текст апелляции:
     [текст, который написал участник]
     
     Примите решение (голосование: X сек):
     [Кнопка: Выпустить] [Кнопка: Не выпускать]
     ```
   - Любой участник сервера может проголосовать
   - Можно переголосовать (последний голос учитывается)

4. **Подсчет голосов**
   - По истечении времени голосования подсчитываются голоса
   - Если голосов "за" >= голосов "против" - заключенный освобождается
   - Если голосов "против" больше - заключенный остается под арестом
   - Результаты отображаются в сообщении с апелляцией

#### Время голосования в зависимости от срока ареста:

Время голосования настраивается в конфигурационном файле через параметр `appeal_voting_durations`.

**Пример настроек по умолчанию:**

| Срок ареста | Время голосования | Доступность апелляции |
|-------------|-------------------|----------------------|
| 30 секунд   | -                 | ❌ Недоступна        |
| 60 секунд   | 15 секунд         | ✅ Доступна          |
| 3 минуты    | 20 секунд         | ✅ Доступна          |
| 5 минут     | 30 секунд         | ✅ Доступна          |
| 15 минут    | 50 секунд         | ✅ Доступна          |
| 1 час       | 2 минуты          | ✅ Доступна          |

Вы можете изменить эти настройки в файле `config.json`, отредактировав параметры `arrest_durations` и `appeal_voting_durations`.

#### Ограничения:

- Одна апелляция на один арест
- Апелляция недоступна для арестов на 30 секунд
- Только арестованный может подать апелляцию
- Голосовать могут все участники сервера, видящие канал голосования

## Обработка ошибок

- Если пользователь не в голосовом канале: "❌ Вы не в голосовом чате!"
- Если в канале нет других участников: "❌ В голосовом канале нет других участников для ареста!"
- Если у пользователя нет необходимых прав: "❌ У вас нет прав для использования этой команды!"

## Устранение неполадок

### Бот не отвечает на команды
- Проверьте, что токен бота правильный
- Убедитесь, что у бота есть права на чтение сообщений
- Проверьте префикс команды в `config.json`

### Бот не может переместить участника
- Убедитесь, что роль бота выше роли участника
- Проверьте права "Перемещать участников"

### Бот не может управлять ролями
- Убедитесь, что роль бота выше всех ролей, которыми он управляет
- Проверьте права "Управление ролями"

### Неправильные ID в конфиге
- Включите режим разработчика в Discord
- Скопируйте ID заново
- ID должны быть числами без кавычек
